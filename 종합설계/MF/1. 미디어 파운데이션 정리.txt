미디어 파운데이션은 두 가지 프로그래밍 모델을 제공한다.
1. 미디어에 이벤트를 입력할 수 있어 스트리밍을 제어할 수 있는 모델
2. 미디어를 읽고 출력하며 데이터를 전송 할 수 있는 모델.

기본요소(헬퍼 객체)
- 속성(Attributes) : 일련의 키-값 쌍으로 객체 안에서 정보를 저장하는 일반적인 방식
- 미디어 타입(Media Types) : 미디어 데이터 스트림의 포맷을 나타낸다.
- 미디어 버퍼(Media Buffers) : 비디오 프레임이나 오디오 샘플과 같은 미디어 데이터 덩어리를 가지고 있으며 객체 간 데이터 전송에 사용된다.
- 미디어 샘플(Media Samples) : 미디어 버퍼에 대한 컨테이너이다. 미디어 샘플은 타임 스탬프와 같은 버퍼에 대한 메타데이터를 포함하기도 한다.

미디어 파이프라인
- 미디어 파이프라인은 미디어 데이터를 생성하거나 처리하는 세 가지 종류의 객체를 포함하고 있다.
- 미디어 소스(Media Sources) : 파이프라인 안으로 데이터를 집어 넣는다. 이 때, 미디어 데이터는 비디오 파일과 같이 로컬에 저장되어 있는 파일이나 네트워크 스트림, 켑처 디바이스에서 얻을 수 있다.
- 미디어 파운데이션 변환(Media Foundaion Transforms;): 스트림에서 데이터를 처리한다 인코더와 디코더가 이걸로 구현된다.
- 미디어 싱크(Media Sinks) : 데이터를 실제 사용한다. 디스플레이에 비디오를 보여주거나, 오디오를 재생하거나 미디어파일로 데이터를 쓰는 작업을 수행한다.

소스 리더, 싱크 라이터
- 기본적인 미디어 파운데이션 구성요소 -> 미디어 소스, 변환, 미디어 싱크를 사용하는 다른 방법이다.

- Source Reader
  - Media Foundation파이프라인을 통과하는 미디어 데이터를 읽거나 수정하기 위해 설계된 구성요소
  - 소스리더는 디코더를 로드하고 미디어 소스와 디코더 간의 데이터 흐름을 관리한다.
  - 이러한 경우 사용할 수 있다.
	* 기본 파일 구조에 대해 걱정하지 않고 미디어 파일에서 데이터를 가져오려고 합니다.
	* 오디오 또는 비디오 캡처 장치에서 데이터를 가져오려고 합니다.
	* 데이터 처리 작업은 시간에 민감하지 않거나 프레젠테이션 시계가 필요하지 않습니다.
	* Media Foundation을 기반으로 하지 않는 미디어 파이프라인이 이미 있고 Media Foundation 미디어 소스를 자체 파이프라인에 통합하려고 합니다.

- Sink Writer
 - 오디오 또는 비디오 파일을 인코딩 하기 위한 구성요소
